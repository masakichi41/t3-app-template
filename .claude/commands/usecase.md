sub-agent「usecase-maker」を用いてissue $1 から新しいユースケースを実装します．
先にユーザーに実装計画を提示し，承認を得てから実装を開始してください．

## 実装の流れ

- [ ] `gh issue view *` でissueの内容を確認する
- [ ] ユーザーに実装計画を提示し，承認を得る
- [ ] 実装を行う
- [ ] ユーザーに実装完了を報告し，PRを提出する
- [ ] ユーザーにコードレビューを依頼する
